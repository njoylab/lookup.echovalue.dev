Features & Capabilities

DNS Enrichment is our intelligent analysis layer that goes beyond basic DNS lookups. It includes:

  • Email Security Analysis - SPF/DMARC/DKIM parsing with security scoring (0-100)
  • MX Provider Detection - Automatic identification of email providers (Google Workspace, Microsoft 365, etc.)
  • TTL Categorization - Classification of TTL values (very-short, short, normal, long, very-long)
  • Security Recommendations - Actionable suggestions to improve your email security posture
  • Quick Access Fields - Top-level convenience fields for easy data access

Note: Enrichment may take up to 60 seconds to complete due to the comprehensive analysis performed.

SSL/TLS Inspection fetches live certificate metadata for domains serving HTTPS traffic, including:

  • Certificate issuer and subject information
  • Subject Alternative Names (SANs)
  • Certificate validity period and expiration date
  • Days until expiry (with color-coded warnings)
  • TLS protocol version and cipher suite
  • Self-signed certificate detection

You can even download a calendar reminder for certificate renewal!

DNS Propagation Checking queries your domain across multiple global DNS servers to verify that DNS changes have propagated correctly. We check:

  • Google DNS - 8.8.8.8, 8.8.4.4
  • Cloudflare - 1.1.1.1
  • OpenDNS - 208.67.222.222
  • Quad9 - 9.9.9.9

Results show consistency percentage and highlight any discrepancies between servers. This is especially useful after making DNS changes to verify they've propagated worldwide.

Reverse DNS Lookup (PTR record lookup) resolves IP addresses back to their associated hostnames. This is useful for:

  • Network reconnaissance and validation
  • Email server authentication
  • Identifying server owners
  • Troubleshooting email deliverability issues

Simply enter an IP address (e.g., "8.8.8.8") and we'll automatically perform reverse DNS lookup to find associated hostnames.

Technical Details

Our real-time validation automatically cleans domain input to ensure accurate DNS lookups:

  • Removes protocol prefixes (http://, https://, ftp://)
  • Removes www. prefix
  • Removes paths and query strings
  • Removes port numbers
  • Converts to lowercase

For example, "https://www.Example.com/page" becomes "example.com"

This ensures you get the DNS records for the actual domain, not a URL.

Default settings (configurable in Advanced Options):

  • Query Timeout: 5000ms (5 seconds)
  • Maximum Retries: 3 attempts
  • Retry Delay: 1000ms (1 second) between attempts

These settings use exponential backoff to handle transient network failures gracefully while respecting DNS server rate limits.

Domain names cannot include spaces. This error appears when you enter something like "my domain.com" instead of "mydomain.com".

Common causes:

  • Copy-pasted text with extra spaces
  • Typing multiple words without removing spaces

Simply remove the spaces and try again.

This validation error appears when the domain doesn't match expected format. Common issues:

  • Missing TLD (e.g., "example" instead of "example.com")
  • Invalid characters (only letters, numbers, hyphens, dots allowed)
  • Consecutive dots (e.g., "example..com")
  • Leading or trailing dots/hyphens
  • Too short (minimum 4 characters like "a.co")

Cloudflare Turnstile is our bot protection. If it won't load:

  • Ensure JavaScript is enabled in your browser
  • Check that Cloudflare domains aren't blocked by ad blockers
  • Try disabling browser extensions temporarily
  • Clear your browser cache and reload the page
  • Try a different browser or incognito mode

Not all domains have all record types. Common scenarios:

  • No MX Records: Domain doesn't handle email
  • No AAAA Records: Domain doesn't support IPv6
  • No TXT Records: No SPF, DKIM, or verification records configured

Make sure you've selected the record types you want to query in Advanced Options. By default, we query A, AAAA, MX, CNAME, TXT, and NS records.
